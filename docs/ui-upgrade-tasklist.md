# ClipLoom UI 升级任务清单

日期：2026-04-08

## 目标

- 先把现有 Win32 设置页从“系统默认控件堆叠感”提升到更清晰、更现代的视觉层级
- 再把主弹窗一起纳入统一视觉规范，减少设置页和主界面的割裂感
- 最后用 WinUI 3 PoC 验证现代 UI 外壳是否值得全面推进

## 当前优先级

1. 完成 Win32 设置页止血升级
2. 统一主弹窗样式基线
3. 抽离设置桥接层，支撑 WinUI 3 设置页 PoC
4. 用 PoC 结果决定是否推进全面迁移

## 任务拆分

### A. Win32 设置页止血升级

- 已完成：统一设置页背景、字体层级、卡片式分组和输入留白
- 待完成：给底部操作按钮补齐更明确的主次状态
- 待完成：为 Tab 页签补更现代的选中态和间距
- 待完成：检查中英文两套文案在 100% / 125% / 150% 缩放下的截断情况

验收标准：

- 设置页首次打开时视觉重心清晰，标题、分组、正文和说明文字层级可区分
- 中英文 UI 没有明显文字裁切、控件重叠或卡片溢出
- 现有设置读写、Apply、Save、Close 行为不回归

### B. 主弹窗视觉统一

- 已完成：统一主弹窗背景、搜索框、列表区和预览区的卡片化基线
- 已完成：把列表项升级为双行信息样式，并统一高亮、pin 标识和快捷数字前缀配色
- 已完成：给空状态、无搜索结果状态补统一的信息样式
- 待完成：进一步优化预览区信息层级，减少当前纯文本元数据块的“调试感”
- 检查键盘导航、回车粘贴、Esc 关闭、Delete 删除的交互节奏是否保持一致

验收标准：

- 主弹窗和设置页看起来属于同一套产品，而不是两套风格
- 搜索结果高亮、选中态和普通态对比足够明显
- 不牺牲当前键盘优先的操作速度

### C. UI 基础设施

- 抽一个 `SettingsViewModel` 或 `SettingsSnapshot` 层，隔离 UI 控件和 `AppSettings`
- 整理颜色、字体、圆角、间距等设计 token，避免样式散落在消息处理代码里
- 为设置页和主弹窗分别列出回归检查清单

验收标准：

- 新 UI 不需要直接读写大量散落的 Win32 控件句柄
- 日后无论继续做 Win32 自绘还是转 WinUI 3，都能复用同一套配置桥接层

### D. WinUI 3 设置页 PoC

- 范围只覆盖 `General` 和 `Appearance`
- 读取现有设置快照，修改后通过显式 `Apply` 提交
- 不接管托盘、热键、剪贴板监听和主消息循环
- 记录启动速度、包体、依赖和焦点行为问题

验收标准：

- PoC 可以独立展示设置页核心布局和交互
- 设置修改能映射回现有 `AppSettings`
- 能清楚回答“值不值得全面迁移到 WinUI 3”

## 风险清单

- Win32 自绘继续深入后，代码维护成本会上升
- WinUI 3 PoC 若过早扩成完整壳层，会把验证变成重写
- 中英文文案和高 DPI 适配是最容易被忽略的细节风险

## 建议执行顺序

1. 先补完 Win32 设置页剩余细节并做人工回归
2. 接着统一主弹窗样式和状态表现
3. 然后抽设置桥接层
4. 最后在 Windows 环境中启动 WinUI 3 设置页 PoC
